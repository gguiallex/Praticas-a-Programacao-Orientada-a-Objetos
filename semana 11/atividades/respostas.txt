3 - a) O usuário recebe a mensagem de erro definida na classe Agencia:
Para deposito: "ERRO: Conta não foi encontrada."
Para saque: "ERRO: Conta não foi encontrada."

b) O usuário recebe a mensagem definida no metodo sacar da classe conta: "ERRO: Saldo insuficiente para realizar o saque." 

c) O usuario recebe a mensagem definida no metodo transferir da classe Agencia: "ERRO: Conta de origem não encontrada."

d) Ao tentar transferir de uma conta válida para uma inexistente, o sistema exibe a mensagem incorreta de "Transferencia realizada com sucesso..." porque o método sacar funciona, retirando o dinheiro da conta de origem, entretanto a tentativa de depositar na conta de destino gera um erro, já que é nula. Esse erro é capturado por um bloco try-catch dentro da classe Conta, que não informa a falha para o Menu. O Menu não recebe nenhum erro e assume que a operação foi um sucesso. O saldo da conta de origem não fica errado, pois o mesmo bloco catch que esconde o erro também reverte o saque, garantindo que o dinheiro não seja perdido.

e) O usuário receberá duas linhas: primeiro a mensagem da NullPointerException seguida pela mensagem "Transferencia realizada com sucesso...". Adicionar um println apenas exibe a mensagem da exceção no console, mas não muda o fluxo de controle. Portanto o programa continua a execução normalmente e imprime a mensagem de sucesso.

f) modificações no arquivo agencia da pasta atividade3.

g) a mensagem exibida é "Transferência realizada com sucesso..." mas continua incorreto. Isso ocorre porque o erro de "saldo insuficiente", gerado pelo método sacar, é capturado por um bloco try-catch dentro do método transferir (na classe Conta), mas não é relançado. Como o erro é "engolido", o Menu não fica sabendo da falha e assume que a operação foi bem sucessida. A correção é adicionar o comando throw e; ao final do bloco catch no método transferir da classe Conta. Isso força a propagação do erro até o Menu, garantindo que o usuário receba a mensagem correta: "ERRO: Saldo insuficiente para realizar o saque".
